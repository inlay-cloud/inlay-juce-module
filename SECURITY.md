# Security Policy

## Reporting a Vulnerability

Please report security issues privately by emailing info@inlay.cloud.

Do not open public GitHub issues for vulnerabilities, licensing bypasses,
activation bypasses, token validation problems, or issues that could weaken
customer license enforcement.

Please include:

- a short description of the issue,
- affected files or versions,
- reproduction steps or proof of concept,
- expected impact,
- your preferred contact information.

We will acknowledge valid reports as soon as practical and follow up with
status when there is a material update.
